아키텍처

3개 컴포넌트 + 외부 의존(MongoDB·Nest·LLM 제공자·임베딩·Tailscale). 전체 다이어그램은 Overview. 처리 흐름은 비동기 인입 + 자정 배치 2축.

sequenceDiagram participant P as 폰(앱) participant B as 백엔드 /ingest participant Q as BackgroundTask participant N as Nest→LLM P->>B: POST /ingest (async_mode=1) B->>B: corpus.append() → vault 정본 B-->>P: stub record (status=processing) 즉시 B->>Q: 큐잉 Q->>N: vision/quick/insight 호출 N-->>Q: 결과 Q->>B: MongoDB record 갱신(status=done) loop 4s 폴링 P->>B: GET /records/{id} B-->>P: status=done + 인사이트 end

차량 출차/주차 상태머신

물리 상태전이는 수집기가, "어디 가에 답했나"는 대화를 가진 백엔드(운행 스레드)가 판정. 비대칭 근거: BT 해제=이미 차서 멀어짐(주차 단독), BT 연결=차 근처일 뿐(물건 꺼내기?) → 200m 게이트로 운전 확정.

stateDiagram-v2 [*] --> 주차중 주차중 --> 운전중: 차 BT 연결 + 세운 데서 200m↑ 이동 → "어디 가?" 운전중 --> 주차중: 차 BT 해제(2틱 디바운스) → 주차 GPS + "어디?/잘 도착했어?" 운전중 --> 주차중: 30분 정지(안전망) → 조용히 리셋
  • 출차: POST /car/departure → "어디 가?" + drive.question_ts 기록
  • 주차: POST /car/parking → question_ts 이후 첫 user 답 조회 → "강남 잘 도착했어?", 없으면 "어디 세웠어?"
  • 재시작 baseline: 현재 BT로 상태만 맞추고 이벤트 억제(가짜 출차/주차 방지)
  • 임계값은 settings.location에서 어드민 튜닝

보안 · 프라이버시 · 관측성

  • 네트워크: Tailscale(WireGuard) 내부망 전제. 공개 노출 안 함.
  • 인증: 기본 무인증. ORACLE_TOKEN 설정 시 미들웨어 토큰 검사(loopback 면제).
  • 프라이버시: SMS 본문은 로컬 LLM만(클라우드 금지). OTP redact. 신호 source 태깅(provenance).
  • 관측성: 수집기가 /collector-status로 WiFi·위치·BT·차상태·로그 보고 → 어드민에서 adb 없이 확인.
  • 백업: vault 일일(14일)·mongo 주간(28일) 자동 회전.

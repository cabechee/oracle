디자인 원칙

  • 에디토리얼/발행물 은유 — 스파인(좌측 시간축 레일), 방주(우측 회잉크 주석), No.N 발행물 느낌.
  • 화자 = 서체 — 유저(Pretendard), 동반자/일기(GowunBatang 명조), 캡션·숫자(Literata).
  • 주홍은 한 곳만 — 화면당 액센트(vermilion) 1곳 원칙(원고지 주홍).
  • 위계 = 여백 — 내부 여백 ≤ 외부 여백. 모든 행간 4pt 격자(수직 리듬).
  • 사진은 인화물 — 흰 마진 + 헤어라인 테두리로 '붙인 사진'처럼.

화면 인벤토리

4탭 + 모달 시스템. 흐름은 유저플로우.

화면위치목적 · 주요 요소
홈 (오늘)Tab 0마스트헤드(날짜)·건강 한 줄·발행물·어제 한 줄·오늘 기록·그날의 오늘
흐름 (대화)Tab 1스파인 타임라인 — record+대화 merge, 사진/음성/텍스트 버블, 동반자 방주, 반응
데스크Tab 2당장 처리(신호)·리마인더·오늘 알림·가계부·오래 못 챙긴 사람
기록 (캡처)Tab 3전체화면 카메라·영상·음성·텍스트 + 동반자 질문 배너
일기 (Digest)모달날짜 리스트 + 마크다운 본문 + '발행'
검색 (Query)모달검색바 + 답변 + 참조 record 카드
색인 (Index)모달펜딩 스레드 환기 + 마스터 인덱스
위치 설정모달추적 토글·장소 목록·WiFi 저장 제안
신호 로그모달brief 타임라인 + 원본 신호 + 분류 탭
온보딩 / 진단 로그첫 실행 / 히든사용법 안내 / 디버그 로그(타이틀 long-press)

컴포넌트 카탈로그

각 컴포넌트: 해부 → 변형 → 상태 → do/don't. 핵심 컴포넌트:

RecordBubble

사진(인화 마진)·음성·코멘트 + 베르 인사이트 + 쿠키 방주 + 반응. 상태: processing / done / failed.

ChatMessageBubble

유저(우측 검정)·베르(좌측 아바타)·쿠키(방주 회잉크). 변형: 일반 / 액션 제안(버튼).

AskBanner

기록 탭 상단 동반자 질문 오버레이(반투명). 화자명 + 질문, 탭 시각 기록.

ActionCard (데스크)

신호 액션. 상태: 대기 / 확인됨(dismiss, 실행취소). 카테고리 색.

RefRecordCard

검색/대화 참조 — 사진 썸네일 + 캡션.

표지 마스트헤드

Literata 디스플레이 날짜 + 발행물 No. 에디토리얼 헤더.
상태 규약 최소 4상태(default·hover/press·active·disabled) + 접근성 focus. 비동기 컴포넌트는 processing/done/failed 3상태를 명시.

인터랙션 · 모션

  • 비동기 인입: 전송 즉시 pending 버블 → 4초 폴링으로 done 전환(자연스러운 채움).
  • 동반자 말걸기: 알림 → 탭 → 기록 탭 진입 + 질문 배너 슬라이드. 흐름엔 Q&A 쌍으로 안착.
  • 반응 피드백: 사진 캡션 아래 "정확해·부족해·틀렸어", 코멘트엔 "좋아·싫어".
  • 플랫폼 가드: 웹은 카메라·녹음 안내만, 수집은 Android 수집기 전담(flutterCollects=false).

핸드오프 노트

  • 토큰 SoT = app/lib/core/design.dart — 디자인 변경은 여기서.
  • 남은 디자인: 필름 질감 · 오늘의 한 줄 · 잔여 화면 · 다크 필름 모드.
  • 수집기 대시보드(별도 네이티브)는 라이트 유틸리티 톤 — 메인 앱 에디토리얼과 의도적으로 분리.

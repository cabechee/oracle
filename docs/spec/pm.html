문제 정의 · 배경

일상의 조각(보고 듣고 받은 것)은 흩어지고 휘발된다. 메모 앱은 입력은 받아도 반응하지 않고, 라이프로그 앱은 자동 수집하되 의미를 엮지 않는다. 받은 문자·부재중·푸시는 쌓이기만 하고 중요한 것을 골라주지 않는다.

Oracle은 의도적으로 던진 것에 LLM 동반자가 즉시 반응하고(디스커버리), 흩어진 조각을 자정마다 회고로 엮으며(기억), 받은 신호를 대신 읽어 우선순위를 잡아준다(비서). 자동 캡처가 아니라 사용자가 선택해 던진 것에만 반응하는 게 핵심 차별점이다.

페르소나

1차 사용자 — 본인 (single-user)

역할 개발자 본인(1인). 언어 한국어. 기기 Android(Z Fold) + iPhone 동시 소지.

맥락 일상을 의도적으로 스냅샷 캡처(자동 라이프로그 아님). 집·작업실·카페 방문과 차량 이동이 잦음. 받은 문자·부재중·결제 알림이 많아 선별이 필요. 하루/주/월 단위 회고 리듬을 선호.

니즈 부담 없는 캡처, 똑똑하지만 짧은 반응, 묵은 백로그에 파묻히지 않기, 차/장소 같은 반복 맥락을 기억해주는 것.

베르 — 메인

쿠키 — 서브

오목눈이 새 · 사용자를 '오빠'로 부름

반말, 발랄·장난스러움(기분 나쁘지 않게). 극도로 짧게 한 문장, 길어도 두 문장. 깊은 분석·조언 금지 — 첫인상 가벼운 한마디만.

범위 · 비범위 (Non-Goals)

범위 (In)

  • 1인 개인 사용 (single-user)
  • Tailnet 내부에서만 접근
  • 모델 agnostic — Nest로 어떤 LLM이든
  • 정본은 평문 Vault, Mongo는 캐시
  • 수집=Android, 인터페이스=공통

비범위 (Non-Goal)

  • 다중 사용자·계정·권한 분리
  • 공개 인터넷 노출·SaaS화
  • 자동 라이프로깅(상시 무차별 수집)
  • SMS 본문의 클라우드 LLM 처리 (로컬만)
  • iOS의 신호·위치 수집 (샌드박싱)

가정 · 의존성 · 제약

  • 네트워크: 모든 기기가 Tailscale 한 tailnet. 백엔드는 chocolat.tail575fea.ts.net:8001.
  • LLM: Nest 게이트웨이 가용 가정. 미설정/실패는 graceful(최근순 fallback 등).
  • 프라이버시: SMS 본문은 로컬 LLM만. OTP는 redact. 기본 무인증(ORACLE_TOKEN opt-in).
  • 수집 제약: iOS는 백그라운드 수집 불가 → Android 수집기 전담. FGS는 location 타입(시간제한 회피).
  • 차량: 전기차(테슬라) BT가 잘 안 끊겨 주차 감지에 BT해제 단독 + 안전망 의존.
